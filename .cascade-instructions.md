# Cascade Instructions

## Documentation Updates

When making code changes, ALWAYS:

1. **Review affected documentation files:**
   - README.md (root)
   - packages/framework/README.md
   - packages/test-project/README.md

2. **Update documentation to reflect:**
   - New features/functions added
   - Changed APIs or interfaces
   - New environment variables (update .env.example and README)
   - Updated scripts or commands
   - Modified project structure

3. **Verify documentation accuracy:**
   - All code examples in documentation work
   - All exported functions/classes are documented
   - Version numbers are updated if applicable
   - Installation instructions are current

4. **Specific checks:**
   - When changing `packages/framework/src/index.ts`: Update packages/framework/README.md API section
   - When adding env vars: Update .env.example and relevant README
   - When modifying scripts: Update script documentation in README
   - When changing project structure: Update project structure diagrams

## Project Context

This is a BDD Playwright monorepo with:
- **packages/framework/** - Reusable BDD testing framework (@automite/bdd-playwright-framework)
- **packages/test-project/** - Reference implementation demonstrating framework usage

Technology stack:
- CucumberJS (BDD)
- Playwright (browser automation)
- TypeScript (type-safe code)
- Yarn Workspaces (monorepo management)

## Code Style

- Follow existing TypeScript patterns
- Use existing base classes (BaseWorld, BasePage)
- Maintain consistency with existing step definitions
- Keep changes minimal and focused
